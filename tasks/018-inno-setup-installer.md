\# Task: Inno Setup installer



\## Goal



Create a simple Windows installer for MokoSnap using Inno Setup.



The installer should package the published MokoSnap app and native host into a user-installable application. It should not require admin rights for core functionality.



\## Scope



Allowed changes:



\- installer/

\- scripts/

\- docs/

\- extension/chrome/README.md

\- src/MokoSnap.App/ only if tiny path/startup handling adjustment is required

\- src/MokoSnap.Platform.Windows/ only if tiny installed-layout path resolution adjustment is required

\- src/MokoSnap.Core/ only if tiny path resolution adjustment is required

\- src/MokoSnap.Tests/ only if needed



Forbidden changes:



\- Do not change Chrome capture protocol.

\- Do not implement automatic Chrome extension store installation.

\- Do not write to HKLM.

\- Do not require admin rights.

\- Do not refactor unrelated UI.

\- Do not add unrelated features.



\## Assumptions



\- Local publish output is generated by `scripts/publish-local.ps1`.

\- Installer input comes from `artifacts/publish/MokoSnap/`.

\- App install should be per-user by default.

\- Native Messaging registration remains HKCU-based and is handled by MokoSnap's Chrome Capture Setup UI.

\- The installer does not need to automatically register the Chrome Native Host.

\- The installer does not need to install the unpacked Chrome extension automatically.

\- Inno Setup is installed manually by the developer.



If an assumption is wrong, stop and report it.



\## Implementation Requirements



\### 1. Installer Script



Add:



\- `installer/MokoSnap.iss`



Requirements:



1\. Install MokoSnap App files.

2\. Install MokoSnap NativeHost files.

3\. Use per-user install mode if simple.

4\. Create Start Menu shortcut.

5\. Optionally create Desktop shortcut.

6\. Add uninstall support.

7\. Use app name:

&#x20;  - `MokoSnap`

8\. Use executable:

&#x20;  - `MokoSnap.App.exe`

9\. Do not require admin rights.

10\. Do not write HKLM registry keys.



\### 2. Build Installer Script



Add:



\- `scripts/build-installer.ps1`



Requirements:



1\. Run or require `scripts/publish-local.ps1`.

2\. Locate Inno Setup compiler `ISCC.exe`.

3\. Compile `installer/MokoSnap.iss`.

4\. Output installer to:

&#x20;  - `artifacts/installer/`

5\. Print installer path.

6\. Fail clearly if Inno Setup is not installed.



\### 3. Installed Layout Path Resolution



Ensure Chrome Native Host path resolution can find NativeHost in the installed layout.



Expected installed layout can be one of:



\- `{app}\\MokoSnap.App\\...`

\- `{app}\\MokoSnap.NativeHost\\...`



or another documented layout matching publish output.



Requirements:



1\. Running installed MokoSnap should resolve installed NativeHost exe.

2\. Chrome Capture Setup UI should generate manifest pointing to installed NativeHost exe.

3\. Existing development layout should still work.



\### 4. Startup Registration



Ensure startup registration points to installed MokoSnap.App.exe when running installed app.



Requirements:



1\. HKCU startup registration only.

2\. Use `--minimized` argument if supported.

3\. No admin rights.



\### 5. Documentation



Add or update:



\- `docs/INSTALLER.md`

\- `docs/RELEASE.md`



Document:



1\. Required tool:

&#x20;  - Inno Setup

2\. Build commands:

&#x20;  - `./scripts/verify.ps1`

&#x20;  - `./scripts/publish-local.ps1`

&#x20;  - `./scripts/build-installer.ps1`

3\. Output path.

4\. How to install.

5\. How to setup Chrome capture after install.

6\. Known limitations:

&#x20;  - Chrome extension still loaded manually

&#x20;  - Chrome may need restart after native host registration



\### 6. Tests



Add or update tests for:



1\. Installed-layout NativeHost path resolution if testable.

2\. Startup command generation for installed app path if testable.



Avoid tests that require Inno Setup to be installed.



\## Verification



Run:



\- `./scripts/verify.ps1`

\- `./scripts/publish-local.ps1`

\- `./scripts/build-installer.ps1` if Inno Setup is installed



Manual check:



1\. Run verify.

2\. Run publish script.

3\. Run installer build script.

4\. Install MokoSnap.

5\. Launch MokoSnap from Start Menu.

6\. Confirm tray/background behavior works.

7\. Open Chrome Capture Setup.

8\. Confirm NativeHost path resolves to installed NativeHost exe.

9\. Register Native Host.

10\. Restart Chrome.

11\. Capture Chrome tabs.

12\. Import latest Chrome tabs.

13\. Uninstall MokoSnap.

14\. Confirm installed files are removed.



\## Done Criteria



\- Code builds.

\- Tests pass.

\- Publish script still works.

\- Installer script exists.

\- Installer build script exists.

\- Installer build either succeeds or fails with clear Inno Setup missing message.

\- Installed layout NativeHost path resolution is supported.

\- Docs explain install/release flow.

\- No admin rights required for core app usage.

\- No unrelated files changed.

\- Final response lists changed files.

\- Final response lists verification result.

\- Final response lists installer build result.

\- Final response lists known limitations.
